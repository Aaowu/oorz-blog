# 修改指南

这份文档是给现在这个站用的。

你以后要改站，不用每次都从头想。照这个来就行。

## 平时怎么改

现在这个站有两种改法。

第一种是在网页前台改。

你打开首页右上角那个设置入口，或者用快捷键 `cmd + ,` / `ctrl + ,`。
能在里面改的内容，改完直接点保存。

比如这些都属于前台改：

- 站点标题
- 用户名
- 配色
- 社交按钮
- 壁纸
- 头像
- favicon
- 帽子
- 一些显示开关

如果保存成功，改动会直接写进 Github 仓库。

第二种是直接改仓库文件。

适合改结构、文案、组件、样式、布局这种前台不方便改的东西。

## 常改文件在哪

### 站点基础配置

站点基础配置在：

`src/config/site-content.json`

这里管这些：

- 标题
- 用户名
- 配色
- 背景色
- 当前壁纸
- 社交按钮
- 是否显示雾气
- 是否显示圣诞效果
- 帽子索引

### 首页布局

首页卡片位置和尺寸在：

`src/config/card-styles.json`

这里管这些：

- 每张卡片开不开
- 宽高
- 偏移位置

如果你觉得首页排版不顺眼，就改这里。

### 首页组件

首页主要组件在：

`src/app/(home)`

常见的有：

- `hi-card.tsx`
- `art-card.tsx`
- `social-buttons.tsx`
- `share-card.tsx`
- `calendar-card.tsx`
- `clock-card.tsx`
- `hat-card.tsx`

### 导航

首页左边导航卡在：

`src/components/nav-card.tsx`

### 关于页

关于页内容在：

`src/app/about/list.json`

### 项目页

项目页内容在：

`src/app/projects/list.json`

### 分享页

分享页内容在：

`src/app/share/list.json`

### 图片页

图片页数据在：

`src/app/pictures/list.json`

### 博客内容

博客内容都在：

`public/blogs`

现在这个目录已经被清空了。
以后如果要重新启用文章，就会重新往这里写。

## 如果只是改内容

最简单的方法就是：

1. 打开前台设置
2. 改内容
3. 点保存
4. 等几秒再刷新

如果你开了 Cloudflare 和 Github 自动部署，就等它自动发版。

如果你没开自动部署，就手动执行一次：

```bash
pnpm run deploy
```

## 如果是改代码

本地改完后，通常这样走：

```bash
pnpm install
pnpm run build
pnpm run deploy
```

如果只是确认 Cloudflare 构建能不能过，也可以用：

```bash
pnpm run build:cf
```

## 自定义域名

域名配置在：

`wrangler.toml`

例如：

```toml
[[routes]]
pattern = "example.com"
custom_domain = true
```

改完以后重新部署：

```bash
pnpm run deploy
```

## Cloudflare 那个 Git 自动部署要不要开

不是必须。

你如果平时喜欢自己手动发版，就不用开。

你如果希望每次 push 到 Github 后自动发布，就开。

我更推荐先手动部署确认没问题，再决定要不要接自动部署。

## 现在这个站有几个容易踩的坑

### 头像和 favicon 改了不刷新

以前这里有缓存问题。
现在已经修过了。

如果你改完头像或 favicon，还是没立刻变：

- 先普通刷新
- 还不行就关掉标签页重新打开

### 背景图上传了但看不出来

以前这里也有坑。
现在已经修过了。

如果你上传了背景图：

- 它会自动设为当前背景
- 你可以在设置里关掉底部雾气，这样更容易看清

### 帽子选了不显示

这个以前是因为帽子卡片默认被关掉了。
现在已经打开。

### 保存时报 `update ref failed: 422`

这通常不是你点错了。

意思是你保存时，Github 分支刚好已经被别的保存动作更新了。

现在首页配置保存已经加了自动重试，一般会自己恢复。

如果以后别的页面也报这个错，最稳的做法是：

- 先刷新页面
- 再重新保存一次

## 我建议你以后怎么改

如果只是改站点展示，用前台设置。

如果要改布局、组件、页面结构，改仓库文件。

如果你不确定该改哪，我的建议是先搜名字。

比如你要改“帽子”，就全局搜 `hat`。
要改“壁纸”，就搜 `backgroundImages`。
要改“标题”，就搜 `meta.title`。
